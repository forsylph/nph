# 문서맵

## 1. 목적

이 문서는 현재 기준 구조에서 어떤 역할의 문서가 어디에 있는지 빠르게 찾기 위한 안내문이다.

## 2. 현재 기준 축

- `030.index`
  - 운영 규칙, 문서맵, 용어집, 읽는 순서, process journey, Tech Stack
- `031.front-channel`
  - MiPlatform, 화면 진입, navigation/command, 프론트 이벤트
- `032.framework-core`
  - DevOn, ServiceProxy, PC/UC/EC, XML Query, LCommonDao, TX/Pool
- `033.platform-services`
  - 보안/인증, 공통 연동, 외부 솔루션/패키지
- `035.Biz-medical-Domain`
  - patient journey, EMR viewer, 의료업무/도메인 관점 문서
- `037.runtime-trace`
  - 대표 화면/업무의 실제 실행체인
- `038.fact-todo-reference`
  - reference, fact-check, todo
- `95/953.refactoring-ideation`
  - 리팩토링 타겟, 비교, 개선안

## 3. old Data 사용 원칙

- `old Data`는 보존본이다.
- 기준본이 모자랄 때만 참고한다.
- 새 분석 결과는 현재 기준 구조에 올린다.
