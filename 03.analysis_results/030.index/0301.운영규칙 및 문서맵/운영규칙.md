# 운영규칙

## 1. 목적

이 문서는 `03.analysis_results` 재구성 이후 현재 기준 구조를 운영하는 규칙을 정리한 문서다.

## 2. 현재 기준 구조

```text
03.analysis_results/
├─ 030.index/
├─ 031.front-channel/
├─ 032.framework-core/
├─ 033.platform-services/
├─ 035.Biz-medical-Domain/
├─ 037.runtime-trace/
├─ 038.fact-todo-reference/
└─ old Data/

95.추가 검토 사항 및 계획/
└─ 953.refactoring-ideation/
```

## 3. 규칙

- `030`은 허브로만 사용한다.
- `031~033`은 기반기술 축이다.
- `035`는 의료업무/도메인 축이다.
- `037`은 사례 추적 문서만 둔다.
- `038`은 fact-check, todo, reference를 관리한다.
- 리팩토링 아이데이션은 `95/953`에서 관리한다.
- 과거 조사본은 `old Data`에 보존한다.

## 4. 문장 규칙

- 직접 확인한 내용만 확정 문장으로 쓴다.
- 미확인 항목은 `미확인`, `추정`, `가능성`으로 적는다.
- 과거 추정을 현재 사실처럼 재사용하지 않는다.
