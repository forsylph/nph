# 약어 용어집

## 1. 목적

이 문서는 `Architecture - Framework` 문서에서 자주 나오는 약어, 클래스명, 계층명, 대표 파일군 이름을 빠르게 확인하기 위한 색인이다.

## 2. 정리 방식 제안

이 폴더에서는 아래 두 기준을 같이 쓰는 것이 가장 실용적이다.

1. `구획 순`
- Front Channel
- Business Layer
- Data Access
- Infra / TX / Pool
- Runtime Trace / File Family

2. `빠른 찾기용 약어 순`
- 약어 자체를 바로 찾을 때 사용

이 문서는 우선 `구획 순`으로 정리하고, 마지막에 `빠른 찾기용 약어 목록`을 둔다.

## 3. Front Channel

### Action
- navigation XML에서 `.mhi` URL과 연결되는 실행 이름
- 최종적으로 command 클래스와 연결된다

### Command
- front channel에서 business 계층으로 들어가는 서버 진입 클래스
- 실제 로직의 종착점이 아니라 `PC/UC/EC`로 내려가는 출발점인 경우가 많다

### Interceptor
- 요청 전후에 공통 정책을 적용하는 계층
- 예: 로그인 체크, 업로드 처리, 권한 체크

### LActionContext
- request/response와 실행 context를 담는 공통 컨텍스트 객체

### LCommandEngine
- action 문자열을 받아 command를 실행하는 front dispatch 엔진

### LServiceDelegator
- `LServiceProxy` 아래에서 실제 method interception을 수행하는 위임 계층
- CGLIB `MethodInterceptor` 구현체

### LServiceProxy
- service proxy 생성 진입점
- `TxServiceUtil` 경로와 같이 보게 된다

### MiplatformConverter
- MiPlatform `Dataset`과 DevOn 내부 `LData / LMultiData`를 변환하는 adapter
- business 로직이 아니라 채널 변환 계층이다

### MiplatformRequest
- MiPlatform 요청 dataset 수신/해석 계층

### MiplatformServlet
- MiPlatform 요청 진입 servlet

### Navigation XML
- `.mhi` URL의 action과 command를 매핑하는 설정 파일

## 4. Business Layer

### CheckLoginMiCMD
- 로그인 action `CheckLoginUser`에 연결되는 대표 command
- front-channel에서 command가 어떻게 business로 내려가는지 보여주는 기준 사례다

### DrgPostRevwMngmPC
- DRG/심사후처리 흐름을 조합하는 PC
- 조회 전용처럼 보이지만 후처리 규칙과 상태 분기를 함께 품고 있다

### EC
- Execution Component
- DB 접근 중심 실행자
- `LCommonDao`를 가장 자주 직접 사용한다

### EdiMngmPC
- EDI 수신/저장 흐름을 조합하는 분기형 PC
- `samFileId + version`에 따라 query family를 선택한다

### PC
- Process Component
- 업무 흐름 조합자
- 여러 `EC/UC` 호출 순서를 묶는다

### PostRevwEC
- `HP_DMS02204M`에서 `DrgPostRevwMngmPC` 아래로 내려가는 대표 EC
- `/hp/dms/hpdmhdmbs/retrieveDrgRevwPtList` 같은 query path를 호출한다

### PrscMngmPC
- `MD_ORD01001P`의 처방 조회/저장/DUR 흐름을 조합하는 대표 PC
- `PrscSaveUC`, `CommonptPrscEC`와 함께 보게 된다

### RecvEdiFileEC
- `EdiMngmPC` 아래에서 실제 query path를 호출하는 대표 EC
- `sQuery + "/retrieveEdiRecvRcpn" + version` 패턴의 수렴점이다

### RetrieveDrgRevwPtListCMD
- `HP_DMS02204M`의 대표 조회 command
- `drgNavi.xml -> RetrieveDrgRevwPtListCMD -> DrgPostRevwMngmPC` 흐름을 잡는 기준점이다

### RetrieveEdiRecvRcpnCMD
- `HP_DMS01303M` 계열의 대표 조회 command
- `clamNavi.xml -> RetrieveEdiRecvRcpnCMD -> EdiMngmPC` 흐름을 잡는다

### RetrievePtOrderCMD
- `MD_ORD01001P` 조회 계열 대표 command
- `ptmdcrNavi.xml -> RetrievePtOrderCMD -> PrscMngmPC` 흐름의 시작점이다

### SavePtOrderPreCMD
- `MD_ORD01001P` 저장/전처방 계열 대표 command
- 저장 흐름이 어디서 business 계층으로 내려가는지 보여준다

### UC
- Usecase Component 또는 공통 업무 계층으로 이해하면 된다
- 공통 규칙, 외부 연계, 세션/보조 시나리오를 품는 경우가 많다

## 5. Data Access

### AutoDAO
- DevOn의 `LCommonDao` 계열 표면 API를 묶어 부르는 표현
- ORM이 아니라 XML Query + JDBC 기반 접근 구조에 가깝다

### LCommonDao
- NPH 업무 코드에서 직접 가장 자주 보이는 DAO 표면 API
- query path를 받아 조회/수정/프로시저 실행을 감싼다

### LQueryMaker
- `LCommonDao` 내부에서 SQL 해석/변환을 담당하는 helper
- NPH 업무 코드에서 직접 잘 안 보이지만, 런타임 내부 중요도는 높다

### Query Path
- `"/a/b/c/queryId"` 형태의 경로 문자열
- `devonhome/xmlquery/a/b/c.xml`의 `statement name="queryId"`와 연결된다

### XML Query
- 실제 SQL statement를 담는 `devonhome/xmlquery/*.xml` 파일군

## 6. Infra / TX / Pool

### DataSource
- DB connection을 얻기 위한 자원 추상화
- 현재 NPH 문맥에서는 JNDI 기반 해석이 가장 강하다

### JNDI
- WAS가 제공하는 datasource/resource 참조 방식

### LJDBCTransactionManager
- JDBC 기반 transaction manager

### LJTATransactionManager
- JTA 기반 transaction manager

### LDataSourcePool
- JNDI / JDBC / DBCP 경로를 수용하는 datasource 추상화 계층

### Pool
- DB connection pool
- 현재 해석 기준으로는 `프레임워크 추상화 + WAS JNDI 자원 사용` 쪽이 더 안전하다

### TxServiceUtil
- service 호출 시 transaction/spec을 적용하는 진입 유틸
- `getTxService`, `getNTxService`, `getJTxService`를 통해 service proxy를 탄다

## 7. Runtime Trace / File Family

### clamNavi
- 청구/EDI 계열 navigation 파일군
- `RetrieveEdiRecvRcpn`, `SaveEdiRecvRcpn` 같은 action을 포함한다

### drgNavi
- DRG/심사후처리 계열 navigation 파일군
- `RetrieveDrgRevwPtList` 같은 action을 포함한다

### hpdmhdmbs
- `HP_DMS02204M`와 연결되는 대표 xmlquery 파일군
- 조회처럼 보여도 심사 후처리 statement가 함께 들어 있다

### hpdmhf*
- EDI 수신 파일군 중 `F` 계열 xmlquery family
- 예: `hpdmhf201`, `hpdmhf402`, `hpdmhf604`

### hpdmhi*
- EDI 수신 파일군 중 `I` 계열 xmlquery family
- 예: `hpdmhi010`

### hpdmht*
- EDI 수신 파일군 중 `N/T` 성격의 xmlquery family
- 예: `hpdmht201`, `hpdmht204`

### MD_ORD01001P
- 조회/저장/DUR/CPS가 과밀하게 한 화면에 얹힌 대표 화면

### mdmdhtord
- `MD_ORD01001P`의 처방 조회/전처방/자동복사와 연결되는 대표 xmlquery 파일군

### ptmdcrNavi
- 처방 화면 계열 navigation 파일군
- `RetrievePtOrder`, `SavePtOrderPre`, `UpdateDurt` 같은 action을 포함한다

### scninfo
- `MD_ORD01001P`의 DUR/규칙점검과 연결되는 대표 xmlquery 파일군

### HP_DMS01303M
- EDI 수신/분기형 PC 사례와 연결되는 대표 화면

### HP_DMS02204M
- 심사 후처리 도메인 파일군이 조회 흐름과 함께 묶인 대표 화면
