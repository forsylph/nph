## 1. 개요

### 1.1 예상 API 구조

InnoRules는 Java 기반 Rule Engine으로, 다음과 같은 패키지 구조를 가질 것으로 추정됩니다:

```
com.innoexpert.innorulesj
├── client/                 # 클라이언트 API
│   ├── IRClient.java       # Rule Engine 클라이언트
│   └── IRClientFactory.java
├── session/                # Rule 세션 관리
│   ├── IRSession.java      # Rule 실행 세션
│   └── IRSessionFactory.java
├── data/                   # 데이터 객체
│   ├── IRData.java         # 입력/출력 데이터
│   └── IRDataSet.java      # 데이터 집합
├── exception/              # 예외 처리
│   ├── IRException.java
│   └── IRRuleNotFoundException.java
└── batch/                  # 배치 연동
    └── initializer/        # 초기화 클래스들
        ├── LoggerConfigurator.java
        ├── JNDIInitializer.java
        ├── UserDefinedFunctionLoader.java
        ├── RuleInitializer.java
        └── RuleClientInitializer.java
```

## 2. 예상 사용 패턴

### 2.1 기본 Rule 실행 패턴

```java
// [⚠️ 추정] PC/EC 클래스에서의 사용 예시
package nph.his.md.opn.otptnrcr.pc;

import com.innoexpert.innorulesj.client.IRClient;
import com.innoexpert.innorulesj.session.IRSession;
import com.innoexpert.innorulesj.data.IRData;
import devonframework.persistent.autodao.LCommonDao;
import devonframework.persistent.data.LData;
import devonframework.persistent.data.LMultiData;

public class OtptMdcrPrprPC {

    /**
     * 수가 계산 Rule 실행 예시
     * [⚠️ 추정] 실제 확인 필요
     */
    public LData calculateFee(LData patientData) {
        try {
            // 1. Rule Engine 클라이언트 획득
            IRClient client = IRClient.getInstance();

            // 2. Rule 세션 생성
            IRSession session = client.createSession("rule");

            // 3. 입력 데이터 변환 (LData -> IRData)
            IRData input = new IRData();
            input.setString("patientId", patientData.getString("pid"));
            input.setString("deptCode", patientData.getString("deptCd"));
            input.setString("drugCode", patientData.getString("drugCd"));
            input.setInt("quantity", patientData.getInt("qty"));

            // 4. Rule 실행
            session.setInputData(input);
            IRData output = session.execute("DRG_FEE_CALCULATION");

            // 5. 결과 변환 (IRData -> LData)
            LData result = new LData();
            result.setString("totalFee", output.getString("totalFee"));
            result.setString("insuranceFee", output.getString("insuranceFee"));
            result.setString("patientFee", output.getString("patientFee"));

            return result;

        } catch (IRException e) {
            // Rule 실행 예외 처리
            throw new LRuntimeException("수가 계산 Rule 실행 오류", e);
        }
    }
}
```

### 2.2 다중 Rule 실행 패턴

```java
// [⚠️ 추정] 여러 Rule을 순차적으로 실행하는 예시
public LMultiData processClaim(LData claimData) {
    IRClient client = IRClient.getInstance();
    IRSession session = client.createSession("rule");

    LMultiData results = new LMultiData();

    // 1. 환자 자격 검증
    session.setInputData(convertToIRData(claimData));
    IRData eligibilityResult = session.execute("PATIENT_ELIGIBILITY_CHECK");

    if (!eligibilityResult.getBoolean("eligible")) {
        throw new LBusinessException("환자 자격 불일치");
    }

    // 2. 수가 계산
    session.setInputData(convertToIRData(claimData));
    IRData feeResult = session.execute("FEE_CALCULATION");

    // 3. 청구 가능 여부 검증
    session.setInputData(feeResult);
    IRData validationResult = session.execute("CLAIM_VALIDATION");

    if (!validationResult.getBoolean("valid")) {
        throw new LBusinessException(validationResult.getString("errorMessage"));
    }

    return convertToLMultiData(validationResult);
}
```

### 2.3 Batch 연동 패턴

```java
// [⚠️ 추정] 배치 모듈에서의 사용 예시
package nph.bat.sp.reader;

import com.innoexpert.innorulesj.client.IRClient;
import com.innoexpert.innorulesj.session.IRSession;
import devonframework.batch.item.reader.AbstractItemReader;

public class RuleBasedItemReader extends AbstractItemReader {

    private IRClient ruleClient;
    private IRSession ruleSession;

    @Override
    public void initialize() {
        // 배치 초기화 시 Rule Engine 연결
        ruleClient = IRClient.getInstance();
        ruleSession = ruleClient.createSession("rule");
    }

    @Override
    public Object read() {
        // 데이터 읽기
        LData inputData = readFromSource();

        // Rule 적용
        ruleSession.setInputData(convertToIRData(inputData));
        IRData result = ruleSession.execute("BATCH_PROCESSING_RULE");

        return convertToOutput(result);
    }

    @Override
    public void close() {
        // 세션 종료
        if (ruleSession != null) {
            ruleSession.close();
        }
    }
}
```

### 3.2 TxServiceUtil 연동 예시

```java
// [⚠️ 추정] TxServiceUtil을 통한 Rule Engine 서비스 호출
public class FeeCalculationPC {

    public LData calculateFee(LData input) {
        // 1. 서비스 프록시 획득
        // [⚠️ 추정] InnoRules 서비스가 TxServiceUtil에 등록되어 있을 가능성
        FeeRuleService service = (FeeRuleService) TxServiceUtil.getNTxService(
            "nph.rules.FeeRuleService"
        );

        // 2. Rule 기반 계산 수행
        return service.executeRule("FEE_CALC", input);
    }
}
```

## 4. 예외 처리 패턴

### 4.1 Rule 실행 예외

```java
// [⚠️ 추정] 예외 처리 예시
try {
    IRSession session = client.createSession("rule");
    IRData result = session.execute("SOME_RULE");
} catch (IRRuleNotFoundException e) {
    // Rule을 찾을 수 없음
    logger.error("Rule not found: " + e.getRuleName());
    throw new LBusinessException("규칙을 찾을 수 없습니다.");
} catch (IRValidationException e) {
    // Rule 유효성 검증 실패
    logger.error("Rule validation failed: " + e.getMessage());
    throw new LBusinessException("규칙 검증 실패: " + e.getMessage());
} catch (IRException e) {
    // 기타 Rule Engine 예외
    logger.error("Rule engine error", e);
    throw new LRuntimeException("Rule Engine 오류", e);
}
```

## 5. 미확인 항목

### 5.1 확인 필요 사항

| 항목 | 확인 방법 | 우선순위 |
|------|-----------|:--------:|
| `com.innoexpert` 패키지 import 실제 여부 | 소스 코드 전체 검색 | 🔴 높음 |
| `IRClient`, `IRSession` 클래스 실제 존재 여부 | JAR 파일 분석 | 🔴 높음 |
| TxServiceUtil에 등록된 Rule 서비스 목록 | `devon-framework.xml` 확인 | 🔴 높음 |
| 배치 모듈에서의 실제 사용 패턴 | `NPH_BAT` 소스 분석 | 🟡 중간 |

### 5.2 검증되지 않은 가설

| 가설 | 근거 | 검증 필요 |
|------|------|:---------:|
| LData -> IRData 변환 필요 | 데이터 타입 불일치 예상 | ✅ 필수 |
| 세션 단위로 Rule 실행 | 일반적인 Rule Engine 패턴 | ⚠️ 권장 |
