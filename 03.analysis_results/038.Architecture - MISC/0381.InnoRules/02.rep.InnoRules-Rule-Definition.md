## 1. 개요

### 1.1 IRL (InnoRule Language) 이란?

IRL은 InnoRules에서 사용하는 도메인 특화 언어(DSL)로, 비즈니스 규칙을 선언적으로 정의하는 언어입니다. Drools의 DRL (Drools Rule Language)과 유사한 구조를 가질 것으로 추정됩니다.

### 1.2 Rule 저장 위치

| 위치 | 설명 | 확인 상태 |
|------|------|:---------:|
| **Database (RULESDB_DEV)** | `LOAD.DEFERRED=NO` 설정으로 DB 저장 추정 | ⚠️ 미확인 (스키마 미확인) |
| **파일 시스템** | `.rule`, `.rules`, `.irl` 파일 | ❌ 미발견 |
| **Java 소스 코드** | Rule 코드 하드코딩 | ✅ 확인됨 |

## 2. 예상 IRL 문법 구조

### 2.1 기본 Rule 구조

```irl
// [⚠️ 추정] IRL 기본 구조 예시
RULE "수가 계산 규칙"
WHEN
    $patient : Patient(age > 0, insuranceType != null)
    $treatment : Treatment(code matches "[A-Z]{3}\d{4}")
THEN
    // 계산 로직
    double baseFee = FeeTable.lookup($treatment.code);
    double insuranceRate = InsuranceRate.get($patient.insuranceType);

    $result.setTotalFee(baseFee);
    $result.setInsuranceCovered(baseFee * insuranceRate);
    $result.setPatientCharge(baseFee * (1 - insuranceRate));
END
```

### 2.2 RuleSet 구조

```irl
// [⚠️ 추정] RuleSet 구조 예시
RULESET "외래수가계산"

IMPORT nph.rules.data.Patient;
IMPORT nph.rules.data.Treatment;
IMPORT nph.rules.data.FeeResult;

GLOBAL FeeTable feeTable;
GLOBAL Logger logger;

// 규칙 1: 기본 수가 계산
RULE "기본 수가 계산"
PRIORITY 10
WHEN
    $patient : Patient(deptCode != null)
    $treatment : Treatment(treatmentDate != null)
    $result : FeeResult()
THEN
    logger.info("수가 계산 시작: " + $treatment.code);
    calculateBaseFee($treatment, $result);
END

// 규칙 2: 보험 적용
RULE "보험 급여 적용"
PRIORITY 20
WHEN
    $patient : Patient(insuranceType in ["건강보험", "의료급여"])
    $result : FeeResult(totalFee > 0)
THEN
    applyInsurance($patient, $result);
END

// 규칙 3: 비급여 항목 처리
RULE "비급여 항목 처리"
PRIORITY 30
WHEN
    $treatment : Treatment(insuranceCovered == false)
    $result : FeeResult()
THEN
    $result.setPatientCharge($result.getTotalFee());
    $result.setInsuranceCovered(0);
END

END RULESET
```

## 4. 예상 NPH 업무 Rule

### 4.1 수가/DRG 계산 Rule

```irl
// [⚠️ 추정] DRG 분류 Rule 예시
RULE "DRG 분류 규칙"
WHEN
    $admission : Admission(principalDiagnosis != null)
    $patient : Patient(age >= 0)
    $procedures : List(size > 0) from collect(Procedure())
THEN
    // 주진단코드 기반 DRG 그룹 결정
    String drgGroup = DRGClassifier.classify(
        $admission.principalDiagnosis,
        $procedures,
        $patient.age,
        $patient.gender
    );

    // DRG 점수 계산
    double drgScore = DRGScoreCalculator.calculate(drgGroup);

    // 급여일수 결정
    int coveredDays = DRGUtil.getCoveredDays(drgGroup);

    insert(new DRGResult(drgGroup, drgScore, coveredDays));
END
```

### 4.2 청구 검증 Rule

```irl
// [⚠️ 추정] 청구 가능 여부 검증 Rule 예시
RULE "청구 가능 여부 검증"
WHEN
    $claim : Claim(
        claimDate != null,
        treatmentEndDate != null,
        patientId != null
    )
THEN
    // 1. 진료일자 유효성
    if ($claim.treatmentEndDate.after($claim.claimDate)) {
        $claim.addError("진료종료일이 청구일보다 늦습니다.");
    }

    // 2. 자격 유효성
    if (!PatientValidator.isEligible($claim.patientId, $claim.treatmentStartDate)) {
        $claim.addError("환자 자격이 유효하지 않습니다.");
    }

    // 3. 중복 청구 확인
    if (ClaimHistory.exists($claim.patientId, $claim.treatmentDate, $claim.treatmentCode)) {
        $claim.addError("중복 청구가 의심됩니다.");
    }
END
```

### 4.3 약국 조제 Rule

```irl
// [⚠️ 추정] 약국 조제 Rule 예시
RULE "약물 복용량 검증"
WHEN
    $prescription : Prescription(drugCode != null, quantity > 0)
    $patient : Patient(age > 0, weight > 0)
THEN
    // 1. 성인/소아 용량 구분
    double maxDose = DrugInfo.getMaxDailyDose($prescription.drugCode);

    if ($patient.age < 18) {
        maxDose = maxDose * 0.5; // 소아는 50%로 제한
    }

    // 2. 1일 최대 용량 검증
    if ($prescription.dailyDose > maxDose) {
        $prescription.addWarning("1일 최대 용량 초과: " + maxDose);
    }

    // 3. 복용 기간 검증
    int maxDays = DrugInfo.getMaxTreatmentDays($prescription.drugCode);
    if ($prescription.daysSupply > maxDays) {
        $prescription.addWarning("최대 처방 일수 초과: " + maxDays + "일");
    }
END
```

## 5. User Defined Function (UDF)

### 5.1 Java UDF 예시

```java
// [⚠️ 추정] user-defined-function.xml에 정의될 Java 메서드
package nph.rules.functions;

import com.innoexpert.innorulesj.function.UDF;
import com.innoexpert.innorulesj.function.UDFParam;

/**
 * 날짜 유틸리티 UDF
 */
public class DateUtilsUDF {

    @UDF(name = "daysBetween")
    public static int daysBetween(
        @UDFParam(name = "startDate") String startDate,
        @UDFParam(name = "endDate") String endDate
    ) {
        // 날짜 차이 계산 로직
        return DateUtil.calculateDays(startDate, endDate);
    }

    @UDF(name = "isWeekend")
    public static boolean isWeekend(
        @UDFParam(name = "date") String date
    ) {
        return DateUtil.isWeekend(date);
    }
}

/**
 * 수가 계산 UDF
 */
public class FeeCalculationUDF {

    @UDF(name = "calculateBaseFee")
    public static double calculateBaseFee(
        @UDFParam(name = "treatmentCode") String code,
        @UDFParam(name = "quantity") int quantity
    ) {
        return FeeTable.lookup(code) * quantity;
    }

    @UDF(name = "applyInsuranceRate")
    public static double applyInsuranceRate(
        @UDFParam(name = "baseFee") double baseFee,
        @UDFParam(name = "insuranceType") String type
    ) {
        double rate = InsuranceRate.getRate(type);
        return baseFee * rate;
    }
}
```

### 5.2 UDF XML 정의

```xml
<!-- [⚠️ 추정] user-defined-function.xml 예시 -->
<user-defined-functions>
    <function name="daysBetween"
              class="nph.rules.functions.DateUtilsUDF"
              method="daysBetween"
              return-type="int">
        <param name="startDate" type="String"/>
        <param name="endDate" type="String"/>
    </function>

    <function name="calculateBaseFee"
              class="nph.rules.functions.FeeCalculationUDF"
              method="calculateBaseFee"
              return-type="double">
        <param name="treatmentCode" type="String"/>
        <param name="quantity" type="int"/>
    </function>
</user-defined-functions>
```

## 6. 미확인 항목

### 6.1 확인 필요 사항

| # | 항목 | 확인 방법 | 우선순위 |
|---|------|-----------|:--------:|
| 1 | 실제 Rule 테이블 스키마 | `RULESDB_DEV` DB 조회 | 🔴 높음 |
| 2 | IRL 파일 실제 존재 여부 | 파일 시스템 검색 | 🔴 높음 |
| 3 | Rule 버전 관리 방식 | `LOAD.VERSION=NONE` 설정과의 관계 | 🟡 중간 |
| 4 | UDF 실제 구현 클래스 | `user-defined-function.xml` 파싱 | 🔴 높음 |

### 6.2 검증되지 않은 가설

| 가설 | 근거 | 검증 필요 |
|------|------|:---------:|
| Rule은 DB에만 저장 | `LOAD.DEFERRED=NO` | ✅ 필수 |
| IRL은 Java-like 문법 | Drools DRL과 유사한 구조 추정 | ⚠️ 권장 |
| 수가/DRG Rule 존재 | 병원 시스템 특성 | ⚠️ 권장 |

## 7. 마이그레이션 시 고려사항

### 7.1 Rule 마이그레이션 전략

| 현재 (IRL) | 마이그레이션 후 | 난이도 | 비고 |
|------------|-----------------|:------:|------|
| `.irl` 파일 | `.drl` (Drools) | 중간 | 문법 변환 필요 |
| DB 저장 Rule | DB 저장 Rule | 낮음 | 테이블 마이그레이션 |
| UDF | Java 함수 | 낮음 | 거의 동일 |
| Decision Table | DMN Decision Table | 중간 | 표준화 가능 |
