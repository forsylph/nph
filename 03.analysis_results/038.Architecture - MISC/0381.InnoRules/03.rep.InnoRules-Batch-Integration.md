## 1. 개요

### 1.1 배치 모듈에서의 InnoRules 사용

InnoRules의 초기화 클래스 패키지명(`com.innoexpert.innorulesj.batch.initializer`)으로 미루어 볼 때, **배치 모듈에서 주로 사용**됩니다.

**실제 사용 확인**:
- `nph/bat/hp/job/OtptClamCrtnJob.java` - 배치 Job에서 Rule 관련 코드 확인
- 주석 처리된 Rule 코드: `#S00000311`, `#S00000607` (2014년도 사용 후 비활성화)
- Rule 사용 여부 체크: 공통코드 `MD994` (보험시사 Rule 사용여부)

**배치 Job의 Rule 호출 흐름**:
```
OtptClamCrtnJob.java
    │
    ├─ ClamCrtnRuleUC.ruleDms()  ← Rule 사용 여부 체크 (MD994 공통코드)
    │       │
    │       └─ ClamChkRuleUC.chkClamRule()  ← 실제 Rule 실행
    │               │
    │               └─ InnoRules Engine
    │
    └─ ClamCrtnUC  ← 청구 생성 (여기서도 Rule 코드 설정 가능)
```

**참고**: 배치 Job 내에서 Rule 사용은 현재 주석 처리된 상태입니다. Rule 사용 여부 체크 로직은 유지되어 있으나, `ModuleInitializer.initialize("innorules")` 호출은 비활성화되어 있습니다.

## 3. 배치 Job 유형별 Rule 사용

### 3.1 청구 자료 생성 배치

```java
// [⚠️ 추정] 청구 자료 생성 배치에서의 Rule 사용
package nph.bat.sp.claim;

import com.innoexpert.innorulesj.client.IRClient;
import com.innoexpert.innorulesj.session.IRSession;
import devonframework.batch.item.processor.AbstractItemProcessor;

/**
 * 청구 자료 생성 프로세서
 */
public class ClaimDataProcessor extends AbstractItemProcessor {

    private IRClient ruleClient;
    private IRSession ruleSession;

    @Override
    public void initialize() {
        // 배치 초기화 시 Rule Engine 연결
        ruleClient = IRClient.getInstance();
        ruleSession = ruleClient.createSession("rule");
    }

    @Override
    public Object process(Object item) {
        ClaimRawData rawData = (ClaimRawData) item;

        // 1. 환자 자격 Rule 검증
        IRData input = new IRData();
        input.setString("patientId", rawData.getPatientId());
        input.setDate("treatmentDate", rawData.getTreatmentDate());

        ruleSession.setInputData(input);
        IRData eligibilityResult = ruleSession.execute("PATIENT_ELIGIBILITY_CHECK");

        if (!eligibilityResult.getBoolean("eligible")) {
            // 자격 불일치 시 스킵
            return null;
        }

        // 2. 수가 계산 Rule 실행
        input.setString("treatmentCode", rawData.getTreatmentCode());
        input.setInt("quantity", rawData.getQuantity());

        ruleSession.setInputData(input);
        IRData feeResult = ruleSession.execute("FEE_CALCULATION");

        // 3. 결과 객체 생성
        ClaimProcessedData processed = new ClaimProcessedData();
        processed.setPatientId(rawData.getPatientId());
        processed.setTotalFee(feeResult.getDouble("totalFee"));
        processed.setInsuranceCovered(feeResult.getDouble("insuranceCovered"));
        processed.setPatientCharge(feeResult.getDouble("patientCharge"));

        return processed;
    }

    @Override
    public void close() {
        if (ruleSession != null) {
            ruleSession.close();
        }
    }
}
```

### 3.2 DRG 분류 배치

```java
// [⚠️ 추정] DRG 분류 배치에서의 Rule 사용
package nph.bat.ipn.drg;

import com.innoexpert.innorulesj.client.IRClient;
import com.innoexpert.innorulesj.session.IRSession;
import devonframework.batch.item.processor.AbstractItemProcessor;

/**
 * DRG 분류 프로세서
 */
public class DRGClassifierProcessor extends AbstractItemProcessor {

    private IRClient ruleClient;
    private IRSession ruleSession;

    @Override
    public void initialize() {
        ruleClient = IRClient.getInstance();
        ruleSession = ruleClient.createSession("rule");
    }

    @Override
    public Object process(Object item) {
        InpatientAdmission admission = (InpatientAdmission) item;

        // DRG 분류 Rule 실행
        IRData input = new IRData();
        input.setString("principalDiagnosis", admission.getPrincipalDiagnosis());
        input.setString("secondaryDiagnoses", admission.getSecondaryDiagnoses());
        input.setString("procedures", admission.getProcedures());
        input.setInt("age", admission.getPatientAge());
        input.setString("gender", admission.getPatientGender());
        input.setInt("los", admission.getLengthOfStay()); // 재원일수

        ruleSession.setInputData(input);
        IRData drgResult = ruleSession.execute("DRG_CLASSIFICATION");

        // 결과 적용
        admission.setDrgGroup(drgResult.getString("drgGroup"));
        admission.setDrgScore(drgResult.getDouble("drgScore"));
        admission.setDrgWeight(drgResult.getDouble("weight"));
        admission.setCoveredDays(drgResult.getInt("coveredDays"));

        return admission;
    }

    @Override
    public void close() {
        if (ruleSession != null) {
            ruleSession.close();
        }
    }
}
```

### 3.3 검증 및 검수 배치

```java
// [⚠️ 추정] 청구 검증 배치에서의 Rule 사용
package nph.bat.mr.verify;

import com.innoexpert.innorulesj.client.IRClient;
import com.innoexpert.innorulesj.session.IRSession;
import devonframework.batch.item.validator.AbstractItemValidator;

/**
 * 청구 데이터 검증기
 */
public class ClaimDataValidator extends AbstractItemValidator {

    private IRClient ruleClient;
    private IRSession ruleSession;

    @Override
    public void initialize() {
        ruleClient = IRClient.getInstance();
        ruleSession = ruleClient.createSession("rule");
    }

    @Override
    public ValidationResult validate(Object item) {
        ClaimData claim = (ClaimData) item;

        // Rule 기반 검증
        IRData input = new IRData();
        input.setString("patientId", claim.getPatientId());
        input.setString("treatmentCode", claim.getTreatmentCode());
        input.setDate("treatmentDate", claim.getTreatmentDate());
        input.setDouble("totalFee", claim.getTotalFee());

        ruleSession.setInputData(input);
        IRData validationResult = ruleSession.execute("CLAIM_VALIDATION");

        ValidationResult result = new ValidationResult();

        // 오류 수준별 처리
        List<String> errors = validationResult.getList("errors");
        List<String> warnings = validationResult.getList("warnings");

        if (errors != null && !errors.isEmpty()) {
            result.setValid(false);
            result.setErrors(errors);
        } else {
            result.setValid(true);
            if (warnings != null && !warnings.isEmpty()) {
                result.setWarnings(warnings);
            }
        }

        return result;
    }

    @Override
    public void close() {
        if (ruleSession != null) {
            ruleSession.close();
        }
    }
}
```

### 5.1 Rule 실행 로깅

```java
// [⚠️ 추정] Rule 실행 로깅 설정
public class RuleExecutionLogger {

    private static final Logger logger = LoggerFactory.getLogger(RuleExecutionLogger.class);

    public void logRuleExecution(String ruleName, IRData input, IRData output, long executionTime) {
        logger.info("[Rule Execution] name={}, time={}ms", ruleName, executionTime);

        if (logger.isDebugEnabled()) {
            logger.debug("[Rule Input] {}", input);
            logger.debug("[Rule Output] {}", output);
        }

        // 느린 Rule 경고
        if (executionTime > 1000) {
            logger.warn("[Slow Rule Detected] name={}, time={}ms", ruleName, executionTime);
        }
    }
}
```
