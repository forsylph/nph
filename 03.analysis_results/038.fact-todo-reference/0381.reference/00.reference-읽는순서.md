# reference 읽는순서

/용어는 [03.약어-용어집.md](../0310.index/03.%EC%95%BD%EC%96%B4-%EC%9A%A9%EC%96%B4%EC%A7%91.md) 를 먼저 보면 빠르다.

이 문서는 `0316.reference` 문서를 어떤 상황에서 참고하면 되는지 짧게 정리한 안내문이다.

reference 문서는 구조 설명의 본문이 아니라, 이미 나온 설명을 확인하거나 근거를 보강할 때 보는 문서다.

## 2. 추천 순서

```mermaid
flowchart LR
    A[기준본 문서] --> B[Reference 가이드]
    B --> C[jar/API/설정 확인]
```

1. [01.Reference-가이드.md](./01.Reference-%EA%B0%80%EC%9D%B4%EB%93%9C.md)
   - reference 문서를 언제 열어야 하는지 보는 첫 문서
2. [02.devon-framework.jar.md](./02.devon-framework.jar.md)
   - `devon-framework.jar`, API 문서, 내부 클래스 실존 확인이 필요할 때 보는 문서

## 3. 무엇을 확인할 때 어떤 문서를 보나

| 확인하려는 것 | 먼저 볼 문서 | 메모 |
|------|------|------|
| `LCommonDao`, `LQueryMaker`가 실제 있는가 | `02.devon-framework.jar.md` | jar/API 문서 기준 실존 확인 |
| `ServiceProxy`, `LServiceDelegator`가 실제 있는가 | `02.devon-framework.jar.md` | API 문서와 jar 엔트리 확인용 |
| DataSource / TX / Pool 관련 클래스가 있는가 | `02.devon-framework.jar.md` | `LDataSourcePool`, `LJDBCTransactionManager` 등 |
| reference 문서를 언제 써야 하는가 | `01.Reference-가이드.md` | 본문과 reference 역할 분리용 |

## 4. 이 폴더를 보는 기준

- 먼저 `0311~0315`의 기준본 문서를 읽는다.
- 구조 설명 중 근거가 필요할 때만 `0316.reference`로 내려온다.
- reference 문서에서 확인한 사실은 다시 기준본 문서로 올라가서 읽는다.
