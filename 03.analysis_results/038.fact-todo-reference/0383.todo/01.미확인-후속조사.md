# 미확인 후속조사

## 1. 목적

이 문서는 현재 `03.analysis_results` 기준본을 유지하면서, 아직 완전히 닫히지 않은 항목만 별도로 관리하기 위한 목록이다.

## 2. 원칙

- 이 문서에 적힌 항목은 아직 확정 문장으로 승격하지 않는다.
- 근거가 더 확보되면 해당 기준본 폴더(`031`, `032`, `033`, `035`, `037`)로 올리고, 여기서는 제거한다.
- 우선순위는 실무 추적 가치 기준으로 둔다.

## 3. 우선순위 A

### A1. `urlPriv` 실제 부착 지점
- 현재 상태
  - `UrlPrivCheckInterceptor` 실존 확인
  - 기본 stack(`defaultStack`, `notLoginCheckStack`, `miUploadStack`)에의 직접 부착은 미확인
- 다음 조사
  - navigation include 구조와 global 설정을 더 타고 들어가 실제 적용 action 확인

### A2. `LJndiDataSource`, `LJdbcDataSource` 내부 구현 세부
- 현재 상태
  - 실존 class는 확인
  - 내부 메서드 구현 세부는 미확인
- 다음 조사
  - jar/API 추가 추적 또는 바이트코드 수준 확인

### A3. `LDataSourcePool`의 실제 pool 소유권
- 현재 상태
  - JNDI / JDBC / DBCP 경로 수용 계층으로는 해석 가능
  - 실제로 프레임워크가 pool을 직접 소유하는지, 주로 WAS wrapper인지 세부 미확인
- 다음 조사
  - pool/free/connection 반환 경로 추가 확인

## 4. 우선순위 B

### B1. `resolveRawSQL`가 쓰이는 실제 화면/경로 확대 확인
- 현재 상태
  - 페이징/특수 경로에서 사용되는 점은 확인
  - NPH 대표 화면별 사용 위치는 추가 조사 필요

### B2. `LQueryMaker` 내부 XML 로딩/metadata 적용 상세
- 현재 상태
  - `resolveSQL`, `resolveRawSQL`, `getFetchSize`, `isSetMetadata` 실존 확인
  - 실제 XML loader 호출/metadata 반영 조건은 세부 미확인

### B3. `LServiceProxy` 아래 mediator 선택 규칙
- 현재 상태
  - `LServiceDelegator`, mediator 계층 존재는 확인
  - local/remote/EJB3 중 NPH 실사용 비중은 미확인

## 5. 우선순위 C

## 7. 배치 추가 검토 예정 항목

### 7.1 현재 정리된 결론

- 현재 배치 스케줄 분석은 `devon-batch-scheduler.xml`만으로 닫히지 않는다.
- 현행 운영 구조는 `batchMgr UI + DB 메타데이터 + BatchExecutor.cmd` 조합으로 보는 것이 더 타당하다.
- 따라서 운영 스케줄을 파악할 때는 스케줄러 엔진 XML보다 `navigation/batch/navigation.xml`, `BatchOnlineExecuteCMD -> BatchInfoPC -> BatchInfoUC`, 실행 로그를 함께 봐야 한다.

### 7.2 남은 한계

- DB 메타데이터가 없어서 현재 운영 JobGroup/Job의 전체 스케줄 표를 아직 만들 수 없다.
- 특히 실제 배치 관리자 화면에서 관리되는 JobGroup, Job, 실행주기, 대상 Job 매핑은 DB 확인 없이는 완전 복원이 어렵다.
- 로그 기준으로 `HP_BAT01206B`, `HP_BAT01206B01`, override 파라미터, 상태 전이, 예외 구조는 확인됐지만, `HP_BAT01206B01 -> 실제 Java Job 클래스` 매핑은 아직 닫히지 않았다.
- `20260202AMdbwrap.log`와 `devonbatch/core_query.default.xml`, `az/com/olbJobGroupExecTx.xml` 기준으로 `GROUP_ID/JOB_ID` DB 메타 조회 구조는 확인됐다. 따라서 남은 미확인은 메타 조회 구조가 아니라 `개별 JOB_ID의 최종 클래스 연결`이다.

### 7.3 추후 바로 할 작업

1. `navigation/batch/navigation.xml` 기준으로 `batchMgr` 기능 메뉴를 별도 문서로 정리
2. 로그에 찍힌 `HP_BAT01206B` / `HP_BAT01206B01`을 실제 Java Job 클래스와 연결한 사례 문서 1장 작성
3. DB 메타데이터 확보 시 JobGroup/Job/주기/override 구조를 표로 재구성
