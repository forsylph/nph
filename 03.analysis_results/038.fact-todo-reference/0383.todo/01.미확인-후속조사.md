# 미확인 후속조사

## 1. 목적

이 문서는 현재 `03.analysis_results` 기준본을 유지하면서, 아직 완전히 닫히지 않은 항목만 별도로 관리하기 위한 목록이다.

## 2. 원칙

- 이 문서에 적힌 항목은 아직 확정 문장으로 승격하지 않는다.
- 근거가 더 확보되면 해당 기준본 폴더(`031~037`)로 올리고, 여기서는 제거한다.
- 우선순위는 실무 추적 가치 기준으로 둔다.

## 3. 우선순위 A

### A1. `urlPriv` 실제 부착 지점
- 현재 상태
  - `UrlPrivCheckInterceptor` 실존 확인
  - 기본 stack(`defaultStack`, `notLoginCheckStack`, `miUploadStack`)에의 직접 부착은 미확인
- 다음 조사
  - navigation include 구조와 global 설정을 더 타고 들어가 실제 적용 action 확인

### A2. `LJndiDataSource`, `LJdbcDataSource` 내부 구현 세부
- 현재 상태
  - 실존 class는 확인
  - 내부 메서드 구현 세부는 미확인
- 다음 조사
  - jar/API 추가 추적 또는 바이트코드 수준 확인

### A3. `LDataSourcePool`의 실제 pool 소유권
- 현재 상태
  - JNDI / JDBC / DBCP 경로 수용 계층으로는 해석 가능
  - 실제로 프레임워크가 pool을 직접 소유하는지, 주로 WAS wrapper인지 세부 미확인
- 다음 조사
  - pool/free/connection 반환 경로 추가 확인

## 4. 우선순위 B

### B1. `resolveRawSQL`가 쓰이는 실제 화면/경로 확대 확인
- 현재 상태
  - 페이징/특수 경로에서 사용되는 점은 확인
  - NPH 대표 화면별 사용 위치는 추가 조사 필요

### B2. `LQueryMaker` 내부 XML 로딩/metadata 적용 상세
- 현재 상태
  - `resolveSQL`, `resolveRawSQL`, `getFetchSize`, `isSetMetadata` 실존 확인
  - 실제 XML loader 호출/metadata 반영 조건은 세부 미확인

### B3. `LServiceProxy` 아래 mediator 선택 규칙
- 현재 상태
  - `LServiceDelegator`, mediator 계층 존재는 확인
  - local/remote/EJB3 중 NPH 실사용 비중은 미확인

## 5. 우선순위 C

### C1. 대표 화면 3종 외 추가 runtime trace 확대
- 후보
  - 로그인
  - 외래 접수
  - 배치 1건
- 목적
  - 기준본이 특정 화면 3종에만 과도하게 의존하지 않도록 보강

### C2. `0316.reference` 세부 확장 여부 판단
- 현재 상태
  - 최소 가이드는 충분함
  - jar/API 문서를 더 두껍게 풀지는 필요성 검토 후 결정

## 6. DB 스키마 확보 후 즉시 할 작업

### 6.1 InnoRules 저장소 스키마 점검
- 대상 후보
  - `RULESDB_DEV`의 Rule 관련 테이블
  - Rule master / rule set / rule definition / rule_content 계열 컬럼 구조
- 확인 목적
  - `rule_content` 실제 원문 확보
  - Rule 코드와 저장 테이블의 연결 방식 확인
  - IRL 원문이 DB에 어떻게 저장되는지 확인

### 6.2 IRL 실문법 확인
- 대상 후보
  - `rule_content` 컬럼 원문
  - RuleBuilder/저장소 export 산출물 존재 여부
- 확인 목적
  - 현재 문서의 IRL 예시가 실제 문법과 얼마나 가까운지 검증
  - 함수/변수/조건식/결과 구조를 실제 문법 기준으로 재정리

### 6.3 운영 API 재판정
- 대상 후보
  - Rule 테이블의 client/session 관련 메타데이터
  - 배포 라이브러리 버전과 DB 메타정보
- 확인 목적
  - `IRClient` / `IRSession`이 실제 운영 경로에서 쓰였는지 재판정
  - `RulesInterface` / `RuleReq`와의 역할 차이를 저장소/엔진 관점에서 보강
