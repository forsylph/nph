# archive 정책

## 1. 목적

이 문서는 `0319.archive`와 `old/`를 어떤 기준으로 보존하는지 정리한 정책 문서다.

## 2. 보존 원칙

- 루트 `0311~0318`은 현재 기준본이다.
- `old/`는 과거 조사 과정에서 만든 문서를 구조 유지 상태로 보존하는 공간이다.
- `0319.archive`는 앞으로 발생하는 보존본, 정리 전 스냅샷, 더 이상 기준본이 아닌 초안/이전판을 두는 공간이다.

## 3. `old/`와 `0319.archive`의 차이

### `old/`
- 최초 재구성 이전의 문서군
- 과거 폴더 구조를 유지한 상태의 보존본
- 현재 기준본의 직접 비교 대상

### `0319.archive`
- 재구성 이후 추가로 생기는 보존본
- 기준본에서 밀려난 이전판
- 주제는 유효하지만 더 이상 주 문서가 아닌 문서

## 4. 무엇을 archive로 보내는가

- 기준본을 대체하고 남은 이전 버전
- 구조는 맞지만 문체나 정합성이 떨어져 참고용으로만 남길 문서
- 실험용 정리본, 병합 전 초안, 임시 통합본

## 5. 무엇은 archive로 보내지 않는가

- 현재 루트 기준본
- fact-check, todo, glossary처럼 현재 운영에 직접 필요한 문서
- `old/`에서 아직 비교 근거로 써야 하는 원본

## 6. 운영 규칙

1. 새 기준본이 생기면 기존 기준본을 바로 삭제하지 않는다.
2. 비교 가치가 있으면 `0319.archive`로 옮긴다.
3. 구조 재편 이전 원본은 `old/`에서 유지한다.
4. README, 문서맵, fact-check에는 archive 문서를 기준본처럼 링크하지 않는다.
