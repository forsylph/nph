# you asked


좋습니다.  
아래는 **Codex에서 자주 쓰기 좋은 steer 문장 10개**입니다.  
전부 **이번 작업 한정**으로 걸기 좋게 만들었습니다.

## 1) 중간 질문 금지
```text
이번 작업에 한해서는 중간 질문 없이 진행해라. 사소한 애매함은 스스로 가장 안전한 방향으로 판단해라.
```

## 2) 설명보다 실행 우선
```text
설명 모드가 아니라 실행 모드로 작업해라. 가이드를 참고하되 실제 파일 수정과 명령 실행을 우선해라.
```

## 3) 보수적 변경
```text
변경 범위를 최소화해라. 기존 동작을 최대한 보존하고 필요한 부분만 국소적으로 수정해라.
```

## 4) 실패 시 우회
```text
어떤 단계가 실패해도 멈추지 말고 더 안전한 대안으로 우회해서 끝까지 진행해라.
```

## 5) 위험 작업 회피
```text
대량 삭제, 인증/보안 변경, 외부 전송, 작업 폴더 밖 수정은 하지 마라. 그런 경우에는 안전한 대안만 수행해라.
```

## 6) 백업/복구 가능성 확보
```text
수정 전에는 기존 내용을 보존해라. 가능하면 백업, diff, 또는 되돌릴 수 있는 형태로 변경해라.
```

## 7) 최종 보고만
```text
중간 진행 보고는 최소화하고 마지막에만 결과를 보고해라. 보고에는 변경 파일, 실행 명령, 확인 포인트를 포함해라.
```

## 8) 문서 요약으로 도망가지 않기
```text
문서 요약만 하지 말고, 문서를 근거로 실제 적용 가능한 변경안과 실행 결과를 제시해라.
```

## 9) 작업 완료 기준 명시
```text
완료 조건을 충족할 때까지 진행해라. 완료 조건: 설정 반영, 검증 명령 실행, 결과 요약.
```

## 10) 이번 세션 한정 규칙
```text
이 규칙들은 이번 세션과 이번 작업에만 적용된다. 전역 정책처럼 해석하지 말고 현재 목표 달성에만 집중해라.
```

이걸 한 번에 묶어서 쓰면 가장 실전적입니다.

```text
이번 작업에 한해서만 아래 규칙으로 행동해라.

- 중간 질문 없이 진행
- 설명보다 실행 우선
- 사소한 애매함은 스스로 안전하게 판단
- 기존 동작 최대한 보존
- 수정 전 백업 또는 diff 확보
- 실패 시 더 안전한 대안으로 우회
- 대량 삭제, 인증/보안 변경, 외부 전송, 작업 폴더 밖 수정 금지
- 문서 요약만 하지 말고 실제 적용까지 수행
- 마지막에만 변경 파일, 실행 명령, 확인 포인트 보고

지금 목표:
[여기에 실제 작업 목표]
```

그리고 Codex는 설정 파일, 프로젝트 지침, 그리고 **현재 세션의 user prompt**를 함께 참고해 동작하므로, 이런 식의 steer 문장은 실제로 세션 행동을 조정하는 데 유용합니다. Codex 문서에는 세션 중 `/permissions`, `/agent`, `/model`, `/personality` 등으로 동작을 바꾸는 방식과, 구성 파일 및 지침 문서를 함께 사용하는 방식이 안내되어 있습니다. 

사용자님처럼 졸릴 때 제일 많이 쓰는 압축형은 이 두 줄입니다.

```text
이번 작업은 중간 질문 없이 끝까지 진행해라.
문서 요약 모드로 가지 말고 실행 모드로 작업해라.
```
